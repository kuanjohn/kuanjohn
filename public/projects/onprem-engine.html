CloudiMesh On-Prem Engine

On-premises appliance that syncs vSphere inventory, runs provisioning jobs, and bridges datacenter hypervisors to the CloudiMesh cloud.

GitHub

What it does

The CloudiMesh On-Prem Engine is a datacenter appliance that collects live vSphere inventory (an RvTools-equivalent), runs provisioning jobs against local hypervisors, and syncs a governed cloud catalog back to the CloudiMesh SaaS — with outbound-only traffic scoped to a single tenant and site.

Problem solved

Bridge SaaS portal intents to on-prem hypervisors — collect live vSphere inventory, run provisioning jobs, and sync a governed catalog back to CloudiMesh, all with outbound-only traffic.

How it works

  • RvTools-equivalent inventory sync from vCenter
  • Cloud catalog mapping to CloudiMesh tenant
  • Provisioning job queue with attempts & sync status
  • Multi-hypervisor drivers (VMware, Nutanix, Huawei)
  • Outbound-only engine API key per tenant/site
  • Signed engine updates with snapshot and rollback
